Для запуска создайте файл .env в корневой директории и пропишите переменные по аналогии с example.env

Далее запустите докер демон и выполните команду 

``doocker compose up --build``

после этого методы будут доступны для просмотра в по адресу 

`http://localhost:{app_port}/docs`

Установите в вверху справа ваш APP_API_KEY заголовок для доступа к api

Как проверить что все методы отрабатывают согласно заданию:

 - 1 список всех организаций находящихся в конкретном здании:
/api/v1/organizations?building_id={id здания}&include_subactivities=false

 - 2 список всех организаций, которые относятся к указанному виду деятельности
/api/v1/organizations?activity_id={id деятельности}&include_subactivities=false

 - 3 список организаций, которые находятся в заданном радиусе/прямоугольной области относительно указанной точки на карте. список зданий
   - по радиусу:
   /api/v1/organizations/within_radius?latitude={}&longitude={}&radius_km={}

   - в заданном квадрате:
/api/v1/organizations/within_rectangle?min_lat={}&max_lat={}&min_lon={}&max_lon={}

 - 4 вывод информации об организации по её идентификатору
/api/v1/organizations/1

 - 5 искать организации по виду деятельности. Например, поиск по виду деятельности «Еда», которая находится на первом уровне дерева, и чтобы нашлись все организации, которые относятся к видам деятельности, лежащим внутри. Т.е. в результатах поиска должны отобразиться организации с видом деятельности Еда, Мясная продукция, Молочная продукция.
проставьте параметр include_subactivities=true 
/api/v1/organizations?activity_id=3&include_subactivities=true

 - 6 поиск организации по названию
/api/v1/organizations?name={символы для поиска}&include_subactivities=false

 - 7 ограничить уровень вложенности деятельностей 3 уровням
ограничено в методе вложенного поиска
А также на уровне модели
